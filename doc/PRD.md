# PRD — Core Source Model

## Status
Draft (inputs pending from owner).

## Problem Statement
Growth breaks repeatability: systems multiply, departments drift, data fragments, decisions slow, and accountability disappears. The Core Source Model restores a single source of truth so the business can scale without added overhead or complexity.

## Target Audience
**Primary persona:** CEOs/owners of $1M+ revenue businesses with existing tools/teams and cross‑department friction.

**Out of scope (disqualifiers):** experimenting with business model, seeking tools without structure, unwilling to share data, seeking discounts or quick fixes.

## Objectives
- Establish a single source of truth for operational decisioning.
- Reduce manual reporting and conflicting data.
- Improve clarity of ownership and accountability.

## Scope
### In Scope
- 4‑phase engagement: Diagnosis → Alignment → Integration → Stabilization.
- CEO‑only decision maker.
- Structural correction, not tool-first delivery.

### Out of Scope
- “Tooling only” engagements without structural alignment.
- Education-only or inspirational consulting.

## Phases & Outcomes (Product Requirements)
1. **Diagnosis**
   - Map real operational flows and data divergence points.
   - Identify decision blind spots and accountability gaps.
   - Output: diagnosis map + list of conflicts/noise.

2. **Alignment**
   - Define authoritative source of truth.
   - Define ownership and decision flows.
   - Remove redundant processes/tools.
   - Output: operating model + ownership matrix + metrics definitions.

3. **Integration**
   - Connect workflows to enforce the agreed structure.
   - Enforce data consistency + automation only where it reduces friction.
   - Output: integrated workflow map + automation spec.

4. **Stabilization**
   - Governance rules, decision ownership, tool boundaries.
   - Output: governance charter + change control rules.

# PRD — Website Bold Refresh (Home Only)

## Status
Draft (home page scope only).

## Objective
Make minecoregroup.com feel deeply bold with larger titles, less text, a clear scrolling chronology, and navigation that behaves like a funnel. Ensure booking is reachable from anywhere within 3 clicks.

## Scope
**In scope:** Home page only.
**Out of scope:** Other pages, SEO, content marketing, non‑home navigation changes beyond funnel behavior.

## User Outcomes
- Immediate clarity of the Core Source Model in the hero.
- Chronological narrative as the user scrolls (diagnosis → alignment → integration → stabilization → booking).
- Booking CTA accessible within 3 clicks from any section.

## Functional Requirements
1. **Deep bold hero**
   - Primary headline significantly larger and higher contrast.
   - Subcopy reduced to 1–2 short lines.

2. **Chronological scroll structure**
   - Sections follow a strict sequence and are visually numbered or stepped.

3. **Funnel‑style navigation**
   - Navigation items map to scroll steps rather than broad pages.
   - Primary CTA persistent (book/review) across sections.

4. **3‑click booking rule**
   - From any section, booking reachable within 3 clicks.

## Non‑Functional Requirements
- Maintain fast perceived load and minimal cognitive load.
- Keep typography and spacing consistent with a bold, premium identity.

## Acceptance Criteria
- Hero headline uses “deep bold” styling with visibly larger type than current.
- Page reads as a step‑by‑step chronology.
- User can reach booking from any section within 3 clicks.
